正则表达式介绍

使用正则表达式，能够对字符串进行匹配。

生活中的实际用例

1、注册账号是判断密码输入是否符合要求，如需要英语字母大小写。

2、身份验证时判断身份证是否符合格式。

…

正则表达式规则

限定符

?：0或1次

*：任意多次

+：1次以上

{}：自定次数或范围

[]：自定义字符或范围

^[]：不出现方括号里的内容

元字符

\d：数字字符，等价于[0-9]

\w：英文字符、数字、下划线

\s：空白符、Tab字符、换行符

\b：单词字符的边界

\D：非数字字符

\W：非单词字符

\S：非空白字符

.：任意字符，除换行符。如.+表示匹配任意字符一次以上。若要匹配.本身，则需要输入\.

^：匹配行首，如^a只会匹配行首的a

$：匹配行尾，如a$只会匹配行尾的a

什么是git

git是一个免费、开源的分布式版本控制系统。

与分布式对应的是集中式，最为流行的是svn。

集中式：

优点：所有设备连接一个中央服务器，从中央服务器上下载代码进行编辑，编辑完上传到服务器即可。

缺点：中央服务器坏掉了就什么都没有了，如果网络出问题也无法下拉代码。

分布式：

优点：除了服务器有代码，每个设备也有完整的代码副本。各个设备之间可以互相同步。

缺点：难 代码保密性差

git的3个工作区域

git分为三个工作区域：工作区（本地工作目录）、暂存区、本地仓库

工作区：Working Directory，即电脑上看到的文件夹。

暂存区：Staging Area，用来保存即将提交的内容。

本地仓库：Local Repository，通过git init创建的仓库，包含了完整的项目历史和元数据，即存储代码和版本信息。

git的类比

工作区：生产车间

暂存区：货车

本地仓库：仓库

生产车间生产的产品先堆在货车上，当堆了一些产品，货车发车将产品一起送到仓库进行存放。

工作区写的代码先通过

需要注意的是，这个类比有一个不太准确的地方，当货车将产品运到仓库后，货车就空了，而当暂存区的代码提交到本地仓库后，暂存区的内容不变。

文件的4种状态

文件有四种状态，分别是未跟踪、未修改、已修改、已暂存

未跟踪：Untrack，新创建的文件，还未被git管理。

未修改：Unmodified，已经被git管理，但是文件的内容还没有修改。

已修改：Modified，已经修改，还未添加到暂存区。

已暂存：Staged，已经修改并且添加到暂存区。

添加和提交文件

值得注意的时，“添加”对应着文件从工作区到暂存区，“提交”对应着文件从暂存区到本地仓库。

tmux是什么

tmux是终端复用器(terminal multiplexer)的缩写，能够用来管理终端，使用户能够在使用多个终端的时候将各个终端管理得井井有条而不至于找不到想要操作的终端。

tmux的三个层级

session（会话）、window（窗口）、pane（窗格）。

概况来说：会话可以被放在后台运行，在一个会话中可以创建很多平行的窗口，在一个窗口中可以创建很多贴满窗口的窗格。

将窗格从最大化恢复：再按一次Ctrl+b z。

关闭窗格：Ctrl+b x。

与关闭窗口相同，tmux会询问你是否确认关闭该窗格。

可以通过将窗口中的每一个窗格关闭的方式来关闭一个窗口。

查看所有的窗口、窗格

查看所有的窗口、窗格：Ctel+b w。

此时可以通过键盘的上下左右来看每个窗口、窗格。

tmux的使用：会话

断开当前会话

断开当前会话：Ctrl+b d。

此时可以看到tmux最下面的绿色任务栏不见了，这并不是tmux被关闭了，而是被放在了后台。通过终端的提示detached也可以知道这只是“断开连接”，并不是关闭。

查看会话数量

在bash终端输入tmux ls。

可以看到，当前创建了一个会话，这个会话的编号是0。

创建新的会话

在bash终端中输入tmux即可，这样会按照数字编号创建新的会话。

如果输入

则会创建新的名为name的会话。

连接断开的会话

连接断开的会话：tmux a -t ‘数字’。这里的a是attach的缩写，也可以将a替换为attach。

如果想要连接最近断开的会话，可以直接输入tmux a。

补充

建议将默认的prefixCtrl + b改掉，因为在很多软件中Ctrl + b也有别的用处。

更换博客主题的曲折过程

发表于 2024-01-31 更新于 2024-02-02 本文字数： 739 阅读时长 ≈ 3 分钟

今天在更换博客主题的过程中很不顺利，一开始想要更换next、butterfly主题怎么都无法成功更换，后来又尝试跟着butterfly官方的文档不知道怎么地就成功了（当然最后还是换上了next主题），我对一开始无法成功更换主题，后面成功更换的原因有三个猜测。

1、在主题配置文件中输错了想要更换的主题名字，或者是“theme：”后面的空格输入错了。（说实话这个可能性比较小，因为最开始我尝试更换了不止一种主题都失败了）

2、在更新主题配置文件后没有关闭git软件，这使得git软件读取到的主题配置文件没有更新（这是我随便猜的）。

3、当我对更换主题不抱有希望时，我看到了一篇教程是跟着butterfly官方文档进行更换主题的，便也自己试着跟着butterfly官方文档来更换主题，一个没有见过的图标显示在了git bash中，我意识这一次更换主题应该是要成功了。

我随即更换了next主题，也成功了。

对于第三个猜测，我猜测是因为在跟着butterfly官方文档的时候使用的git命令对我的博客的某些属性进行了变更，使得更改主题的功能被激活了。（但是我也觉得这个原因的可能性比较小，因为在跟着butterfly官方文档时，我只输入了git clone -b master https://github.com/jerryc127/hexo-theme-butterfly.git themes/butterfly这一行命令）

这个猜测让我想起来”左耳朵耗子“陈皓说过的：学习技术，最好看第一手资料。

虽然一开始无法更换主题，后来能够更换主题这件事情的原因还是使我感到有些好奇，但是无论如何，最终博客的主题都更换成功了，这是令我开心的。我将这次的经历记录在了博客，或许在以后再遇到类似的问题能够有些启发呢。
