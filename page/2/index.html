PCB阻抗匹配学习笔记

发表于 2024-03-27 本文字数： 1.3k 阅读时长 ≈ 5 分钟

为什么高速pcb要阻抗匹配

高速PCB需要阻抗匹配，使信号不会反射，能够通畅到达目的地。

为什么高速PCB需要阻抗匹配，而低速PCB不需要阻抗匹配？因为低速PCB的信号频率低，波长长，阻抗不匹配造成的反射对信号影响小，而高速PCB信号频率高，波长短，阻抗不匹配造成的反射对信号影响很大，反射会造成信号严重失真。

或者也可以换一个理解的角度，之所以有特性阻抗，是因为传输线有寄生电阻、寄生电容、寄生电感，而我们知道电容和电感在交流电中起作用，且交流频率越高，作用越明显，所以对于高速pcb，特性阻抗对电路信号传输的影响就会变大，所以一般在高速pcb中要考虑阻抗匹配。

特性阻抗和电路原理学习的阻抗不一样

电路原理学习的阻抗是将电路看做二端口网络得到网络的阻抗。

特性阻抗是传输线由于存在寄生电阻、寄生电容、寄生电感而存在的阻抗，可以理解为这个传输线各点的特性，这意味着当我们说这个传输线的特性阻抗是多少欧姆的时候，我们不需要考虑传输线的长短（即，特性阻抗与传输线长短无关）。就像是一个铜导线的材质是铜，它的硬度等物理特性由铜这个材质、加工工艺等内容决定，但是与导线的长度无关。

由于特性阻抗和阻抗都是描述电路特性的物理量，都与电压、电流的比值有关，并且单位都是欧姆，两者之前存在许多相似性，所以容易使人搞混，但是我们应该知道特性阻抗和电路原理学习的阻抗不一样。

阻抗匹配联系高中物理

阻抗匹配就是让信号源、传输线、负载的阻抗相同，以达到信号顺利传递的目标。

我想起了在高中物理当中，有一类题目是给定一个存在内阻的电压源，如何使负载功率最大。可以通过不等式或者求导得到当负载电阻与电压源内阻相等时，负载功率最大。

负载功率最大不就可以看做是信号（能量）顺利传递嘛。

从这个视角来说在高中我们就接触过了电阻匹配这个概念。

“特性阻抗”和“差分线”这两个概念之间没有必然联系

在学习高速pcb设计时，“特性阻抗”和“差分线”这两个概念往往同时出现，使人以为这两者是绑定在一起的，实际上不是这样的。

特性阻抗是一种性质，可以说任何传输线都有特性阻抗这个性质。

差分线指的是两根传输差模信号（又称差分信号）的线。在高速pcb中，差分线往往是一定会出现的。但是这不意味着差分线只在高速pcb中出现，在许多对传输信号抗噪声要求较高的地方都有差分线的存在，最令人熟知的就是RS485标准。

通过上面的表述，就不难知道为什么高速pcb中总是把“特性阻抗”和“差分线”放在一起说了，都是为了信号能够以高质量传输。

PCB阻抗如何设置

在高速PCB中，一个信号往往使用两条线进行传输，这两条线称为阻抗对、差分对、差分阻抗对。

像HDMI、USB这些设备都有规定的阻抗，需要在设计PCB的时候对走线进行设置，如设置差分对的线宽和间距，以满足阻抗匹配的要求。

使用阻抗计算工具，我们可以通过输入pcb层数、厚度、铜厚、阻抗要求等内容来计算出需要设计怎么样的线路（如果是单端走线，就会告诉你要设计怎么样的线宽；如果是差分走线，就会告诉你要设计怎么样的线宽和线距）。

阻抗匹配不仅可以通过走线来实现，也可以通过放置元器件来实现。

差分线等长与阻抗的关系

差分线等长与阻抗无关，之所以要等长是为了使两个差分线的差分信号相位相同，方便两个差分信号做减法。

TVS瞬态电压抑制器学习笔记

发表于 2024-03-21 更新于 2024-03-28 分类于 电子技术 本文字数： 1.4k 阅读时长 ≈ 5 分钟

什么是TVS

TVS（Transient Voltage Suppressor，瞬态电压抑制器）是一种特殊的二极管，分双极型和单极型两种，双极型的TVS管具有双向稳压的功能。

当被保护电路瞬间出现浪涌脉冲电压时，双向击穿二极管能迅速齐纳击穿，由高阻状态变为低阻状态，对浪涌电压进行分流和箝位，从而保护电路中各元件不被瞬间浪涌脉冲电压损坏。（摘自百度百科）

TVS的实现原理

齐纳二极管（即稳压二极管）具有单向稳压功能，TVS的实现原理是将两个齐纳二极管的阳极串联起来，从而达到双向的稳压功能。

TVS管和稳压二极管的不同之处

稳压二极管一般不会讲响应时间，更多的是关注稳压值，但是对于TVS二极管不一样，它的特性决定着它拥有不同的作用，TVS二极管具有瞬态抑制高能量作用，因此要求在较短时间内把较高能量瞬间吸收，因此这个时间很短，达到纳秒级别。

从功率来看，稳压二极管的功率比TVS管的功率要低。TVS管的瞬时功率很高。

关键参数

1、截止电压VRWM（Stand-off Voltage）

截止电压指的是不引起TVS管工作的最高工作电压。在VRWM下，TVS是不工作的，即不导通。换一句话，被保护电路的最高工作电压必须小于VRWM，否则将会导致TVS工作导致电路异常。

2、击穿电压VBR（Breakdown Voltage）

击穿电压，指的是在接近发生雪崩的电流条件下测得的TVS两端的电压。

3、钳位电压VC（Clamping Voltage）

钳位电压，指的是施加规定波形的峰值脉冲电流IPP时，TVS两端测得的峰值电压。钳位电压VC应该小于被保护电路的损坏电压。

4、峰值脉冲电流IPP（Peak Pulse Current）

峰值脉冲电流是衡量TVS在电路保护中抵抗浪涌脉冲电流及限制电压能力的参数。对于相同型号TVS，在相同IPP下的VC越小，说明TVS的钳位特性越好。同型号的TVS，IPP 越大，耐脉冲电流冲击能力越强。

5、峰值脉冲功率PPPM（Maximum peak pulse power）

峰值脉冲功率用来衡量TVS能够承受的最大功率，PPPM=VCxIPP，PPPM需要大于电路中可能出现的浪涌能量功率。

选型规范

  1. VRWM，明确TVS的最大反向工作电压，需要保证被保护信号的工作电压略小于VRWM，VRWM=信号工作电压x(1.1~1.2倍系数)。
  2. VC，VC帮助我们判断TVS管的钳位电压是否能和后级电路适配。需要保证VC不超过后级电路的最高承受电压，VC<Vmax(后级电路的最高承受电压)。
  3. IPP，IPP用于判断TVS对瞬时脉冲能量的耐受程度，越大越好。

工程实践

我使用过的TVS有SMF12CA，是我在绘制USB与CH340模块的时候使用的，CH340可以使用上位机的USB线进行供电，为了防止瞬时电压过高损坏CH340，于是增加了一个SMF12CA于USB线的供电与地之间。

SMF12CA介绍

通过翻阅数据手册可以知道，SMF12CA的特性如下：

  • 截止电压VRWM：12V

  • 击穿电压VBR：13.3V-14.7V

  • 钳位电压VC：19.9V

  • 峰值脉冲电流IPP：10.1A

  • 峰值脉冲功率：200W

值得一提的是，在翻阅数据手册的过程中，我发现了SMF12CA中“12”就表示这个TVS管的截止电压是12V。该系列TVS管都以SMF+数字+字母构成，数字是多少说明TVS管的截止电压是多少，字母是CA表示是双极型TVS，字母是A表示是单极型TVS。

Chisel是什么

Chisel语言是一个硬件设计语言（HDL），Chisel是基于Scala语言的一门语言。可以将Chisel看做是Scala的一个拓展库。因此，学习Chisel必须先学习Scala。

Chisel诞生地是UCB（这不难想象，毕竟risc-v就诞生于UCB

Scala是什么

Scala是一门面向对象编程语言。它在运行前被编译成和Java一样的字节码，然后使用JVM运行，也正是因为这样，Scala和Java能够无缝衔接。

Scala语言能够很方便地进行扩展，因此很多DSL（domain-specific language，领域专用语言）基于Scala进行开发。

使用Chisel进行开发的大致流程

许多人对使用Verilog进行开发很熟悉：写Verilog–>综合–>仿真。

实际上使用Chisel进行开发和使用Verilog进行开发流程是相似的。

写Chisel–>firrtl编译器将Chisel文件转换成firrtl文件–>转换成Verilog–>综合–>仿真。

因此使用Chisel进行开发对于熟悉使用Verilog进行开发的人来说并不会感到难受。

正是因为Chisel需要转换成Verilog再生成最终的电路，所以Chisel并不是HLS，而是HDL。

使用Chisel的好处

既然要使用Chisel，那么它一定有比Verilog好的地方。

  1. 面向对象（SystemVerilog的面向对象只针对验证阶段）。
  2. 安全（使用Chisel写的代码转换为Verilog不会生成锁存器）。
  3. 可综合（使用Chisel写的代码转换为Verilog后全部可综合）。
  4. 敏捷。

异步复位指的是无论时钟是否到来，只要复位信号有效就复位。

C语言：内存的各个部分

发表于 2024-03-16 分类于 C语言笔记 本文字数： 1.4k 阅读时长 ≈ 5 分钟

这篇博客记录了我和文心一言的交流。

我：

C语言中，全局变量是放在堆上还是栈上？

文心一言：

在C语言中，全局变量既不是放在堆上也不是放在栈上。全局变量通常存放在静态存储区（Static Storage Area）或者称为程序的数据段（Data Segment）中。这个区域在程序开始执行前就已经分配好，并且在整个程序的执行期间都一直存在。

以下是几种不同变量的存储位置：

  1. 全局变量和静态变量：它们存储在静态存储区（Static Storage Area），这个区域在整个程序的执行期间都保持其值。

  2. 局部变量（非静态）：当函数被调用时，局部变量在栈上分配空间。当函数返回时，这些空间会被自动释放。

  3. 动态分配的内存：使用 malloc、calloc 或 realloc 等函数分配的内存位于堆上。这部分内存的生命周期由程序员通过 free 函数来控制。

  4. 常量：常量通常存储在代码段（Code Segment）或只读数据段中，具体取决于编译器和操作系统。

  5. 函数参数：当函数被调用时，传递给函数的参数通常也存储在栈上。

所以，全局变量的存储与栈和堆无关。它们有自己的存储区域，并且它们的生命周期与程序的执行周期相同。需要注意的是，不同编译器和操作系统可能会有些许差异，但上述描述是C语言标准中对于变量存储的一般理解。

我：

char *str = “hello”;创建的str放在了上面所述的哪个地方？

文心一言：

在C语言中，当你写下char *str = "hello";时，str是一个指针变量，它会被分配在栈上（假设这是在一个函数内部定义的局部变量）。而字符串字面量"hello"则存储在程序的只读数据段（也称为文本段或代码段）中。

这里的关键是理解指针和它所指向的数据之间的区别：

  • str是一个指针变量，它存储的是字符串字面量"hello"的地址。这个指针变量本身是在栈上的。
  • 字符串字面量"hello"是一个常量字符串，它存储在程序的只读数据段中。这意味着这个字符串在程序执行期间是不能被修改的。

因此，当你执行char *str = "hello";时，你实际上是在栈上创建了一个指针变量str，并将它初始化为指向存储在只读数据段中的字符串字面量"hello"的地址。

Dotfile

alias（别名）

为什么在bash中输入ll就可以达到ls -lah相同的效果，正是因为有alias。alias负责进行命令映射。

需要注意的是，alias只接收一个参数，因此=的左右没有空格。

如果想要知道一个命令是从哪个命令映射来的，可以直接输入alias "命令"来知道。

每当终端被关闭后，终端就忘记了这些映射。于是，需要一个配置文件记住这些映射。Linux中的配置文件通常被称为Dotfile，因为这些文件名的开头都是一个点。

无论是在配置vim、tmux还是其他软件时，都是在这个软件对应的dotfile里进行配置，当软件开始运行时，会自动读取配置文件对软件进行配置。

于是，如果能够将alias的映射添加到bash的配置文件中，每次打开bash，bash就会自动将映射记住。
